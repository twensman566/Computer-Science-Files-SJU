Lab 11: Lists and iterators—managing a list of names

In this lab, you will create a simple list manager that allows you to build and maintain a list of names.

As usual, begin by copying the folder for this lab from the common lab directory into your own directory.

Part 1: Extending the HTLUtilities class

The program you'll be creating needs to be able to build a list of names, add more names later, find names, remove names and display the list. In order to do all this, the program will use a HeadTailList<String>, supported by static methods in the HTLUtilities class. Some of the methods you'll need are not yet present in the class, so your first job is to implement them.

The first method you need is one that builds a list of names. The HTLUtilities class already has two methods that build a list of integers, but no methods for building a list of strings. For our purposes, it's probably better to use the signal form, since we can pick a signal string that won't represent a name. Use the readIntListSig() method as a model for a readStringListSig() method; implement the version with a prompt string parameter that is used in the call to readLine().

Adding names is almost trivial if you don't mind adding to the front of the list. All you need to do is construct a new LinkedHeadTailList with the new element as the head and the existing list as the tail, so you don't need to add any new methods to HTLUtilities for this function. HTLUtilities already contains a cull() method that you can use to remove an item from a list.

Finding a name in the list will require a new method. In this case, all we want to do is determine whether a particular name is in the list or not. You can implement a generic find() method on the same pattern as the count() method, but this method two parameters—the list and the element to be searched for—and it returns a boolean. There are three cases: if the list is empty, the method should return false; if the head element equals the element being searched for, it should return true; otherwise, it should return the result of searching the tail of the list for the element. Note that this recursive method actually has two base cases; also note that you must use the equals() method to compare elements.

Finally, we need to be able to display the elements of a list. Implement a generic method that prints the elements, one per line. If the list is empty it should print the message “EMPTY LIST”; otherwise it should print the head element and then recursively print the tail of the list, only if the tail is not empty. (Why?)

You may write a simple test program for your methods if you wish, or you may use the program in Part 2 to test the methods. Do show your code to the TA or lab instructor before you go on.

Part 2: A simple names manager

Write a simple ConsoleProgram that builds and maintains a list of names (Strings). Use the init() method to initialize any instance variables and put the code that operates the program in the run() method. Any operations you do on the list should use methods already available in the HeadTailList interface and the HTLUtilities class.

The program should begin with an appropriate introductory message and then build the initial list. Once the list is built, it enters a command interpreter loop that continues until a “quit” command is issued. The loop should have the following shape:

    do {
        // statements comprising the body of the loop
    while (!cmd.equals("quit"));

The body of the loop begins by displaying a menu of the available commands. The allowed commands should be “add”, “find”, “remove”, “display” and “quit”. The user then enters a command. The program uses an if–else if–else structure to execute the command, comparing the entered command to the target command in each condition part.

For the “add”, “find” and “remove” commands, the program prompts the user for the associated name and then executes the appropriate methods to accomplish the command. In the case of the “add” command, the new name should only be added if it is not already present, so you'll need to use the find() method here; print a confirmation message stating either that the name was added or that it was already present. The “find” command simply determines whether the name is present or not and prints an appropriate message. The “remove” command should first determine whether the name is present or not, only calling the cull() method to remove it if it is there; it should print a message indicating which of the two cases occurred.

The remaining two commands are simple: the “display” command just calls the appropriate method in the HTLUtilities class; the “quit” command does nothing since it's only a signal to exit the loop and end the program. The final else clause is reached if the user enters a command that isn't recognized; in this case, print a suitable error message.

Test your program thoroughly and then show it to the TA or lab instructor.
