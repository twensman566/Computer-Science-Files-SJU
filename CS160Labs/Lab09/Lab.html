Lab 9: Graphical arrays—bullseyes and bowling pins, revisited

Part 1: a new BowlingPins class

Using your Bullseye class as a model and the Lab 5 BowlingPins class for guidance, create a new BowlingPins class. There should be two constructors: the first has no parameters and calls the second (using this()), passing it DEFAULT_NUM_ROWS, which should be defined to be four; the second should take one parameter with the number of rows. The size of the array should be the total number of pins, which is: 

      numRows + numRows−1 + … + 2 + 1
    = numRows * (numRows + 1) ⁄ 2

You'll want to save the number of rows in an instance variable, but you can use the array length if and when you need to check the total number of pins

Add color changing methods similar to the ones in the BoxArray class.

Again use the tester from Lab 5 and/or the tester for the BoxArray as a model to create a tester for your bowling pins. Be sure to test different numbers of rows and different combinations of color and fill changes.

Part 2: a Bowling simulator

Write a graphical program that simulates one frame of a bowling game. We'll use slightly simplified rules, but you can use the actual rules if you prefer. In a typical frame, a bowler gets one or two chances to knock down all the pins. If the first throw knocks down all the pins, it's called a strike, and the player scores 10 plus the number of pins on the next two throws (which will be in the next frame or two if there's another strike). If the second throw knocks down all the pins, it's called a spare, and the player scores 10 plus the number of pins on the next throw (in the next frame). Otherwise, the player scores the number of pins knocked over on the two throws of the frame.

Your simulation should start with a set of standing bowling pins and an array pinsStanding which is initialized to true for all the pins; this much can be set up in the init method. Use the mouseClicked method to initiate a throw. On each throw, generate a number between zero and the number of standing pins, inclusive, to represent the number of pins knocked over on this throw; if the number of pins knocked over is less than the number of pins still standing, randomly select that many of the remaining pins, and mark them as eliminated by unfilling them and setting the corresponding value in pinsStanding to false; at the end of the throw, report the score for the frame so far with a GLabel at the top or bottom of the canvas. The frame ends after the first throw if it is a strike or else after the second throw. If the frame is a strike, report that with the score, and generate a random number between zero and 20 to add to the frame's score; if the frame is a spare, report that, and generate a random number between zero and 10 to add to the score.
