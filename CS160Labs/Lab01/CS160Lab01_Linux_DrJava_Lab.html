As you complete each command, you need to understand what it does. If you need help, remember to ask a TA or a professor.

  1. Start by using the pwd command to make sure you are in the Lab01 directory. If you are not in this directory, get there by using the cd command. You can get there with one command if you recognize that “~/CS160/Lab01” is one of the names of the directory.

  2. Use ls to see what files and directories are in the directory. Next use the tree command to see the subdirectory structure. Compare the resulting diagram to the structure of the Mammals hierarchy above.

  3. Use mkdir to create a subdirectory of Lab01 called Hal. Verify that the new directory was created by using the ls command.

  4. Use cd to change to the directory Mammals, and verify that you're in the Mammals directory using pwd.

  5. List the files and directories in Mammals using ls; then lists the contents of the Cats subdirectory by typing ls Cats.

  6. Change to the Cats directory using cd and display the contents of the directory. (By now you should begin to know which commands do what.)

  7. List only the files with names containing the string “cat” by typing the command ls *cat*. The ‘*’ character is a wildcard that can stand for any sequence of zero or more characters. Compare the filtered file list with the complete list. Which files were eliminated by the filter and why?

  8. Display the contents of the file kittycat by typing more kittyCat.

  9. Change to the directory above the current working directory by typing cd .. . (Recall that “..” is a name for the parent of the current directory.)

  10. Copy the 2cats file to the Haldirectory using a single cp command. Work out names relative to the current working directory for the file to be copied and the directory it is to be copied to before actually typing the command. (Hint: Assuming the current working directory is Mammals, 2cats is in the Cats sub-directory, and Hal is a sub-directory of the working directory's parent directory.) Verify that the file has been copied correctly by listing the contents of both the Hal and Cats directories.

  11. Change the working directory back to Lab01, and verify the command with pwd.

  12. Create a new Puppydirectory as a sub-directory of the Hal directory without changing the working directory. (The name of the new directory is “Hal/Puppy” relative to the working directory.)

  13. Copy all of the files from Dogs into the Puppy directory using a single cp command. (“Mammals/Dogs/*” names all the files in Dogs directory by using the wildcard character.) List the files in the Puppy directory to verify the copy.

  14. Move the Puppy directory from Halto the Mammals/Dogs directory using the mv command. List the contents of the Hal and Dogs directories to verify that Puppy is now in Dogs and not in Hal.

  15. Now that you have had a chance to use some Linux commands in action, build the following directory structure (just the Critters branch) using your own sequence of terminal commands. Verify that you have built the structure correctly using the tree command.

Part 2: Completing a simple Java program

Open a terminal window if you don't already have one. If necessary, change the current working directory to CS160/Lab01. Verify that you have a file named MouseIndicator.class in the directory; if not, you can copy it from the directory /usr/people/classes/CS160/labs/Lab01 into the current directory, using “.” as the second argument to cp. We'll run this program from the terminal instead of from inside DrJava. Type java MouseIndicator at the prompt. Press and release the mouse a few times in the window that pops up. Notice what this program does: when you press the mouse button in the window, the word “up” becomes black and the word “down” becomes red; when you release the mouse button, the “up” becomes red and the “down” becomes black.

You are given in incomplete version of this program that you are to complete so that it works just like the original. Your assignment here is to complete a program that works in this way. Open the file MyMouseIndicator.java in DrJava and add appropriate statements to make it work as specified. Your instructor will give you information about the statements you need to use either in the class before the lab or in the lab itself.
