Lab 3: Graphical programs & simple computation—the Turtle and the Hare

Objectives

  • To write programs using graphics objects
  • To become familiar with the use of random numbers.

In this lab, you will write a Java program that displays a race between two different objects, an oval and a rectangle, representing a turtle and a hare, respectively. With each mouse click, each racer should move a random distance to the right until one of the racers crosses the finish line. You will do this by modifying the file Race.java.

Setup

Using a terminal window, copy the directory /usr/people/classes/CS160/labs/Lab03 into your CS160 directory. Use the ls command to verify that the directory is there and that it contains one Java file, namely Race.java and a directory called solution. Inside the solution directory, you should find a Java class file named Race.class.

Testing a completed version of the program

To see how the program for this lab should work, go into your terminal window, change into directory CS160/labs/Lab03/solution and enter the command java Race, which causes the JVM to run the program Race.class. You may have to resize the window that comes up to see the whole race course. Then click away until the race is run. The program you create in this lab will do the same thing.

Pre-lab exercise

For the pre-lab, you will set up the initial race with the starting line, finish line, turtle (oval) and hare (rectangle). You accomplish this by declaring instance variables for each of the objects in the scene and then creating and setting up the objects in the init() method.

You can place all constant and instance variable declarations in the area above the init() method.

The init() method should place the objects on the canvas for the beginning of the race, including a vertical start line near the left edge, a vertical finish line near the right edge, a gray square to represent the hare and a green circle to represent the turtle, both placed just to the left of the start line, one a bit above the other.

Notice that the program already has the following declaration: 

    private GOval turtle;

We can create the turtle in the init() method after the first statement as follows: 

    this.turtle = new GOval(30, 20);
    this.add(this.turtle, 10, 10);

Compile and run this code. You should see the outline of a black oval in the upper-left corner. The turtle looks like this because the constructor call new GOval(30, 20), followed by this.add(this.turtle, 10, 10). Together these specify that an oval which is 30 pixels wide and 20 pixels high should be placed 10 pixels left and 10 pixels down from the upper-left corner; the default color for any graphical object is black, and it is also drawn as an outline by default. Experiment with changing the position and size of the oval by varying the parameters to the constructor call until you are satisfied that it looks about the same as the one in the Race program.

Now, change the color of the oval and set it so it's filled in. You can change the color with the setColor() method using the statement this.turtle.setColor(Color.GREEN); just between the two statements you added from above. The setFilled() method: this.turtle.setFilled(true) causes the oval to be filled instead of an outline. Compile and run the program to see the effect of adding these statements.

You can create the hare in a similar way, but use a different shape, location and color for the hare. You'll essentially repeat the four statements you used to create the turtle, with the following changes: instead of the GOval constructor, call the GRect constructor, which takes the same two parameters; use a different location for the hare, but you can probably use the same width and height; choose a different color.

Finally, you need to create the start and finish lines, which are both objects in the GLine class. The GLine constructor takes four parameters. The first two parameters are the x- and y-coordinates in pixels of the starting point of the line relative to the upper-left corner of the canvas; the last two parameters are the coordinates of the ending point of the line. Create the two lines and add them to the canvas, experimenting with the start and end points until you have them where you want them. (A little arithmetic and geometry might be helpful.) You can change the color of the lines with the same named setColor() method, but the setFilled() method does not apply to lines.

Work on this program until it produces a display similar to the one in the Race program, but feel free to be a little creative. Show your working program (complete this far) to the instructor or TA at the beginning of lab.
