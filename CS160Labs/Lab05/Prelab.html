Pre-lab exercise: the Bullseye

Open the files BullseyePlacer.java and Bullseye.java in DrJava. These files contain a template for the Bullseye program, which draws a bullseye similar to the Target logo on the canvas. Using a loop, the program should draw six concentric filled-in circles, with the color alternating between red and black. The biggest should be 120 pixels in diameter, with each subsequent circle decreasing 20 pixels in diameter. Each circle should be added to the Bullseye object being created, as it is created—this is accomplished by two statements.

  1. The first is included for you and is the statement extends GCompound in the Bullseye class declaration. The GCompound class is a convenience provided by the ACM library. It allows you to add objects to other objects like you would the canvas. This way, when the parent object, i.e., the object which other objects are being added to, is finally added to the canvas, all of its child objects will be added automatically.
  2. The second is the add() method which is implemented for the Bullseye class because it extends GCompound. You can use it as follows: this.add(circle, x, y);, where circle is a GOval reference variable and x and y are the x- and y-offsets from the Bullseye object to which circle is being added.

Note that you should only modify the Bullseye.java file; keep the file BullseyePlacer.java unchanged.

Test your program to assure yourself that it works correctly. Add appropriate comments if you have not already done so. Show your working program and code to the lab instructor or TA at the beginning of lab.
