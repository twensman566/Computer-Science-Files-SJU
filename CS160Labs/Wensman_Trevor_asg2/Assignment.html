Assignment 2: Java classes and objects

Problem 1—Library

Write a Java class Student that represents a student. The class should define the following:

  • Private instance variables
    • studentName
    • studentEmail
  • Public constructors
    • a single constructor that takes no parameters and assigns instance variables the literal string "UNASSIGNED"
  • Public instance methods
    • provide accessor and mutator methods for both of the instance variables

Write another Java class LibraryCard that represents a library card for a student. The class should define the following:

  • Private instance variables
    • owner — representing the student owning the card
    • borrowedBookCount — the number of books checked out by the student
  • Public constructors
    • a single constructor that takes no parameters and assigns the value null to owner and 0 to the borrowed books count
  • Public instance methods
    • provide accessor and mutator methods for both of the instance variables
    • override the toString() — this method will return a string representation of the LibraryCard object that it is called on; this string should have the following format:
      Owner name: <card owner's name>
      email: <card owner's email>
      Books borrowed: <card owner's borrowed book count>

Write a third Java class LibraryCardTester that is the main class for the program, i.e., it extends the ACM Program class. The run() method for this class should do the following:

  • Create two Student objects and assign them to two local reference variables student1 and student2, respectively. The students should have the following details:
    • student1 — name: John Doe, email: johndoe@csbsju.edu
    • student2 — name: Dawn Wise, email: dawnwise@csbsju.edu
  • Create three LibraryCard objects and assign them to three local reference variables card1, card2, and card3, respectively. card1 and card2 are owned by John and card3 is owned by Dawn.
  • Check out three books using card1, two books using card2, and five books using card3
  • Print the card information for all three cards — you can do this by implicitly or explicitly calling the toString() method from the LibraryCard class.
